【项目代码架构说明】：

  1.common:业务通用公共方法封装

  2.Execute:测试框架启动入口

  3.Mylogs:框架日志模块（业务逻辑共用）

  4.Logs：框架，业务逻辑运行log汇总

  5.testcases：不同测试类型用例集合
  
  6.config: 提供框架统一配置入口,包含任务入口，log存放路径定义，脚本最大执行时间，
            框架运行次数等参数。





【项目运行方式】：python 命令行参数【配置文件】 任务xml文件
                  配置文件：定义框架启动加载目标
                  xml文件： 组织具体测试用例或者测试集合
